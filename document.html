bot-mua-hang

An AI-powered Shopee shopping bot that reads gift orders from an Excel file and autonomously completes the full checkout flow using a Vision LLM.

Python 3.11+ Patchright (Playwright) Gemma-4-31B-IT (VNG Cloud) FastAPI dashboard shopee.vn

Business context

VNG marketing team sends gifts to users. Previously, the purchasing team manually bought each gift on Shopee — taking up to 60 days. This bot reduces the end-to-end time to 10–15 days by automating the full checkout process, including receiver details, seller notes, VAT invoice, cheapest shipping, voucher selection, credit card payment, and OTP verification.

How it works

  1. Import ordersUpload an Excel file via the dashboard UI. Each row is one gift order with product URL, receiver info, and quantity.
  2. Login to ShopeeClick "Login to Shopee" in the dashboard. The browser opens Shopee's login page. After login, the session is saved to disk.
  3. Click StartThe bot opens each product URL, and the Vision LLM (Gemma-4-31B-IT) analyzes screenshots step by step to drive the checkout.
  4. OTP (if required)When Shopee asks for a card OTP, the dashboard shows a popup — the user types the code and the bot continues.
  5. Status written backAfter each order, the Excel file is updated with success/failed status and the Shopee order ID.

The FastAPI server runs in a background daemon thread. The bot loop runs in the main asyncio event loop. They communicate through AppState — a shared singleton that uses threading.Event objects for cross-thread signalling and asyncio.Queue for WebSocket broadcasts.

Shopee login & session persistence

The bot uses a persistent browser profile stored in ./session/. Once you log in, the session is saved and reused on every subsequent run — no need to log in again.

  1. Start the botpython main.py — browser launches immediately.
  2. Check dashboard headerThe login badge shows ✗ Not logged in.
  3. Click "🔑 Login to Shopee"The browser navigates to shopee.vn/buyer/login.
  4. Log in normallyEnter your phone/email and password. Complete any Shopee captcha manually.
  5. Dashboard updatesWithin 3 seconds the badge turns ✓ Logged in. Session is now saved to ./session/.
Reset session — to log in as a different account, delete the session/ directory and re-run.

Why patchright?

Shopee detects standard Playwright/Selenium via the Chrome DevTools Protocol (CDP) fingerprint at the binary level — even with navigator.webdriver = undefined and stealth JS patches. Patchright patches the Chromium binary itself, removing all CDP detection markers that Shopee's anti-bot system looks for.

The first row must be a header row with the column names below. Column order doesn't matter — the bot reads by name.

Checkout flow (8 steps)

After navigating to the product URL, the Vision LLM analyzes each screenshot and executes one action at a time, following this order:

  1. Select product & quantityChoose the correct variant (size/color if applicable), set quantity, click Mua Ngay (Buy Now).
  2. Receiver informationVerify or change the shipping address to match the order's receiver_name, phone_number, and address.
  3. Seller note (Lời nhắn)Type the configured seller_note (e.g. "Quà tặng từ …") into the seller note field.
  4. VAT invoiceIf invoice.company_name is set in config.yaml, enable the invoice checkbox and fill in company name, tax ID, address, and email.
  5. Cheapest shippingCompare all available shipping carriers and select the one with the lowest price.
  6. Apply vouchersClick Chọn Shopee Voucher and apply the best available discount. If a voucher_code is in the Excel row, use that instead.
  7. Credit card paymentSelect Thẻ Tín Dụng/Ghi Nợ — the saved card on the account is used.
  8. Place order + confirmClick Đặt Hàng. If OTP is required, pause for user input. On success screen, extract the order ID.

OTP handling

When Shopee requires a one-time password to authorize the credit card payment, the LLM returns action="wait_otp". The bot then:

  1. Pauses the checkout loopThe order status changes to waiting_otp.
  2. OTP modal appearsA full-screen modal pops up in the dashboard with an OTP input field (auto-focused).
  3. User enters OTPType the code received by SMS/email, press Enter or click Submit. 5-minute timeout.
  4. Bot continuesThe OTP is typed into the browser and the checkout resumes automatically.

Re-buy

The 🔄 Re-buy button on each order row lets you re-process any order that already has a success or failed status. It:

  • Resets the order status to pending in the Excel file
  • Reloads the order from Excel (so any edits you made are picked up)
  • Queues it for immediate re-processing

The bot stays alive indefinitely after finishing all orders, waiting for re-buy requests — no restart needed.

Module reference

src/agent.py — VisionAgent

Wraps the VNG Cloud OpenAI-compatible API. Encodes each screenshot as a base64 image_url and sends it with the checkout objective to Gemma-4-31B-IT. Parses the structured JSON response into an Action dataclass.

Action typeMeaning
clickClick at (target_x, target_y)
typeClick then type text_to_type, press Enter
scrollScroll up or down by 540px
wait_otpPause and request OTP from user
doneOrder successfully placed — extract order ID
failUnrecoverable error — mark order as failed

src/browser.py — BrowserManager

Manages a patchright persistent browser context. The session_dir stores cookies and localStorage across runs. Key methods: start(), goto(url), screenshot(), execute(action), is_logged_in(), close().

src/shopee_bot.py — ShopeeBot

Orchestrates one order at a time: navigate → screenshot loop → execute actions → handle OTP → write result. Builds a detailed checkout objective string from the order data + config (seller note, invoice, voucher, shipping instructions).

src/excel_handler.py

Three functions: load_orders(path) returns all rows as dicts with _row_index. update_order_status(path, row, status, order_id, note) writes the result columns. update_order_fields(path, row, fields) writes arbitrary column updates (used by the Edit endpoint).

src/state.py — AppState

Global singleton shared between the bot loop and the FastAPI server. Contains the orders list, all event primitives, and broadcast() which pushes the serialized state to all connected WebSocket clients.

Anti-bot strategy

Standard Playwright is blocked by Shopee. Shopee's anti-bot system detects CDP (Chrome DevTools Protocol) fingerprints at the browser binary level — JS-level patches alone are not enough.
Detection vectorHow we handle it
CDP binary fingerprintPatchright patches the Chromium binary before compilation — markers are removed at source level
navigator.webdriverRemoved by patchright (binary level)
--enable-automation flagRemoved via ignore_default_args
Sequential orders (rate limiting)Orders processed one at a time with 2.5s delays between actions
Locale / languageBrowser launched with locale="vi-VN"

Session persistence

Patchright's launch_persistent_context(user_data_dir="./session") saves the full Chromium profile — cookies, localStorage, IndexedDB — to the session/ directory. This means:

  • Shopee login persists across bot restarts
  • Saved credit cards and addresses are available
  • To switch accounts: rm -rf session/ and re-login
  • The session/ directory is gitignored
